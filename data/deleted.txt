# 永久跳过列表 —— sync.ts 会读此文件，对应 memo 不会从 flomo HTML 重新生成
#
# 每行格式：<tab>/<id>
# 空行和 # 开头的行被忽略
#
# 使用场景：
#   从 flomo 删除了某些 memo，但重新导出的 flomo HTML 里仍然包含它们
#   （比如用户下载了一份旧快照，或者 flomo 客户端未同步）。
#   不登记此列表，下一次 pnpm sync 会把它们重新写回 md 文件。
#
# 完整删除流程见 README.md 「删除保护」一节。

# 例：
# qh/qh42
# jg/20260116-182134
